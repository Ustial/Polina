# Сайт с комплиментами

Готовая мобильная вёрстка на HTML, CSS и JS. Под GitHub Pages подходит без танцев с бубном.

## Структура

```text
compliment-site/
├── index.html
├── styles.css
├── script.js
├── data/
│   ├── compliments.js
│   └── photos.js
├── assets/
│   ├── fonts/
│   │   └── README.txt
│   ├── icons/
│   │   ├── ht1.svg
│   │   └── ht2.svg
│   └── images/
│       ├── background/
│       │   └── Back.png
│       ├── photos/
│       │   ├── 1.png
│       │   ├── 2.png
│       │   └── 3.png
│       └── logo.svg
```

## Что куда класть

### 1) Комплименты
Открывай файл:

```js
data/compliments.js
```

Там обычный массив строк:

```js
window.COMPLIMENTS = [
  `Первый комплимент`,
  `Второй комплимент`,
  `Третий комплимент`
];
```

Каждая строка это один комплимент.  
Если нужен перенос строки внутри текста, просто нажми Enter прямо внутри шаблонной строки:

```js
`Ты умеешь делать день лучше.
Серьёзно. Это редкий скилл.`
```

### 2) Фотки
Клади изображения в папку:

```text
assets/images/photos/
```

Потом пропиши их в файле:

```js
data/photos.js
```

Пример:

```js
window.PHOTOS = [
  { src: 'assets/images/photos/1.png', alt: 'Фото 1' },
  { src: 'assets/images/photos/2.png', alt: 'Фото 2' },
  { src: 'assets/images/photos/3.png', alt: 'Фото 3' }
];
```

Важно: браузер на GitHub Pages не умеет сам читать содержимое папки.  
Поэтому фотки надо не только закинуть в папку, но и добавить в `photos.js`.

### 3) Фон, логотип, сердечки
Замени эти файлы своими, сохранив имена:

- `assets/images/background/Back.png`
- `assets/images/logo.svg`
- `assets/icons/ht1.svg`
- `assets/icons/ht2.svg`

## Как вставить шрифты нормально

Я не кладу сами файлы шрифтов, потому что это обычно лицензируемая история.

### Что сделать
1. Подготовь web-версии шрифтов:
   - `woff2`
   - по желанию `woff`

2. Положи их в папку:

```text
assets/fonts/
```

3. Лучше использовать такие имена:
   - `DrukCyr-Bold.woff2`
   - `DrukCyr-Bold.woff`
   - `HelveticaNeueCyr-Medium.woff2`
   - `HelveticaNeueCyr-Medium.woff`

4. В `styles.css` уже есть готовые `@font-face`.  
   Если названия файлов будут другими, просто поправь пути.

### Где брать web-формат
Если у тебя шрифты уже есть в `otf` или `ttf`, переведи их в `woff2`.  
Это можно сделать через любой нормальный конвертер шрифтов или через Figma/дизайн-пайплайн, если у вас это уже собрано.  
Главное: используй только те шрифты, на которые у тебя есть право.

## Как запустить локально

### Вариант 1. Самый нормальный
В VS Code поставь расширение **Live Server**.

Потом:
- открой папку проекта
- нажми правой кнопкой на `index.html`
- выбери `Open with Live Server`

### Вариант 2. Просто открыть файл
Можно открыть `index.html` напрямую в браузере, но лучше всё же через Live Server, чтобы сразу ловить реальные пути и поведение как на хостинге.

## Как выложить на GitHub Pages

1. Создай репозиторий на GitHub
2. Закинь в него содержимое этой папки
3. Открой `Settings`
4. Перейди в `Pages`
5. В `Build and deployment` выбери:
   - **Source** → `Deploy from a branch`
   - **Branch** → `main`
   - **Folder** → `/root`
6. Сохрани

Через короткое время GitHub даст ссылку на сайт.

## Где что править по стилю

### Основные цвета и размеры
В `styles.css` в блоке `:root`

Там лежат:
- `--brand`
- `--card-bg`
- `--content-width`
- `--button-width`
- `--radius-card`
- тени и прочее

Если захочешь дожать пиксель-перфект, почти всё правится там.

## Маленький совет
Когда подменишь реальные шрифты, логотип и свои ассеты, сайт сразу станет выглядеть заметно ближе к макету.  
Сейчас внутри лежат рабочие демо-заглушки, чтобы ты мог сразу запустить проект и не смотреть на белую пустоту.
